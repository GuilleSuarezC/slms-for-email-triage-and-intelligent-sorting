## 1. Real spam/ham dataset (binary Track A)

- **Format:** a single CSV of ~194k rows with a text column and a spam/ham label.
- **Source dataset (Kaggle slug):** `meruvulikith/190k-spam-ham-email-dataset-for-classification`
  (this is the exact slug hard-coded in the Track A script).
- **Placement:** you do **not** place it manually. `src/classical_ml/binary_spam/spam_classifier_pipeline.py`
  calls `kaggle.api.dataset_download_files(...)`, unzips it, and reads the CSV.
- **Obtain:** install the `kaggle` client and provide credentials, either in
  `~/.kaggle/kaggle.json` or via `KAGGLE_USERNAME` / `KAGGLE_KEY` (see
  `.env.example`); then run the Track A script â€” it downloads and caches the data.
- **License:** the dataset's license is governed by its Kaggle page and was **not
  resolved** in the source project. Verify and comply with that license before
  redistributing any part of it. This repository does **not** redistribute it.

## 2. LoRA adapter and merged model (Track B â€” LoRA)

These are **regenerated** by the LoRA pipeline; they are not committed (the
adapter alone exceeds GitHub's 100 MB file limit, and the merged model is ~15 GB).

Expected output locations (relative to `src/lora/`, per `config.yaml` `paths:`):

- Adapter: `src/lora/outputs/final_adapter/` (`adapter_model.safetensors`, `adapter_config.json`)
- Merged model: `src/lora/outputs/merged_model/` (`model.safetensors`, tokenizer, `config.json`)

Regenerate (needs an NVIDIA GPU and `requirements-lora.txt`):

```bash
cd src/lora
ln -sf ../../data/email_dataset.csv email_dataset.csv   # dataset where config expects it
# To reproduce the published 70/30 result, first set data.test_size: 0.30 in config.yaml
python scripts/prepare_data.py     # -> outputs/train.csv, outputs/test.csv, outputs/labels.json
python scripts/train_final.py      # -> outputs/final_adapter/
python scripts/merge_lora.py       # -> outputs/merged_model/  (optional; for LM Studio/GGUF)
python scripts/evaluate_test.py    # -> outputs/test_metrics.json, ...
```

The published reference metrics for this stage are in `results/lora_holdout/`.

## 4. SLM weights (zero/few-shot and LoRA base)

- The models `qwen/qwen2.5-vl-7b` (zero/few-shot) and `Qwen/Qwen2.5-7B-Instruct`
  (LoRA base) are **not** stored here. They are served by a local **LM Studio**
  instance and/or downloaded from Hugging Face at run time. Install/load them in
  LM Studio (or via `transformers`) following the README requirements.
